# Limbing Landing Page Architecture

## 1. 목적

이 문서는 Good Dyno 팀이 만드는 Limbing 랜딩페이지 제작을 위한 설계 문서다.

랜딩페이지의 목적은 완성된 앱을 설명하는 것이 아니라, 클라이밍 영상을 찍고 공유하는 사용자에게 "이 서비스가 내 문제를 해결할 수 있겠다"는 감각을 빠르게 전달하고 사전 신청 또는 관심 등록으로 이어지게 만드는 것이다.

## 2. 서비스 한 줄 정의

Limbing은 같은 암장, 같은 난이도, 같은 문제를 푼 사람들의 클라이밍 영상을 한곳에 모아보는 영상 기반 클라이밍 기록 서비스다.

## 3. 랜딩페이지 타겟

초기 타겟은 클라이밍 영상을 반복적으로 촬영하고, 인스타그램 부계정이나 스토리로 풀이 과정을 공유하며, 막히는 문제를 풀기 위해 다른 사람의 영상을 찾아보는 사용자다.

숙련도보다 행동이 중요하다.

- 암장에서 본인 등반 영상을 자주 찍는다.
- 찍은 영상을 인스타그램, 스토리, 릴스, 단톡방 등에 올린다.
- 같은 문제를 다른 사람이 어떻게 풀었는지 궁금해한다.
- 내 풀이 영상과 성장 과정을 나중에 다시 보고 싶어한다.
- 클라이밍과 무관한 지인보다 클라이밍하는 사람들에게 영상을 보여주고 싶어한다.

## 4. 랜딩페이지 핵심 메시지

### 메인 메시지

같은 문제를 푼 사람들의 클라이밍 영상을 한곳에.

### 보조 메시지

Limbing은 흩어진 클라이밍 영상을 암장, 난이도, 문제 기준으로 정리해 내 기록과 다른 사람의 풀이를 함께 볼 수 있게 해준다.

### CTA 문구 후보

- 사전 신청하기
- 내 암장 등록하기
- 출시 알림 받기
- 클라이밍 영상 습관 공유하기

첫 버전에서는 `사전 신청하기`를 기본 CTA로 사용한다.

## 5. 사용자가 느끼는 문제

랜딩페이지는 기능을 길게 설명하기보다, 사용자가 이미 겪고 있는 불편을 먼저 건드려야 한다.

### 강조할 문제

- 영상은 찍지만 갤러리, 인스타그램 부계정, 스토리, 단톡방에 흩어진다.
- 같은 암장, 같은 난이도, 같은 문제를 푼 사람의 영상을 찾기 어렵다.
- 고난이도 문제를 풀 때 참고할 풀이 영상을 찾으려면 인스타그램을 오래 뒤져야 한다.
- 내 풀이 기록은 쌓이지만 암장, 난이도, 문제 기준으로 다시 보기 어렵다.

### 랜딩페이지에서 피할 표현

- "클라이밍을 취미로 즐기는 모든 사람"
- "입문자부터 고수까지"
- "종합 클라이밍 플랫폼"

위 표현은 너무 넓어서 서비스의 날카로운 차별점이 흐려진다.

## 6. 페이지 구성

### 1. Hero

첫 화면에서 제품의 정체성과 사용 맥락이 바로 보여야 한다.

구성 요소:

- 앱 이름: Limbing
- 헤드라인: 같은 문제를 푼 사람들의 클라이밍 영상을 한곳에
- 서브카피: 흩어진 등반 영상을 암장, 난이도, 문제 기준으로 정리하고 내 성장 기록까지 확인하세요.
- CTA 버튼: 사전 신청하기
- 보조 CTA: 서비스 컨셉 보기
- 시각 요소: 모바일 앱 피드 mock 또는 문제별 영상 카드 mock

Hero의 시각 요소는 추상 그래픽보다 실제 앱처럼 보이는 화면 mock이 좋다. 사용자가 "아, 클라이밍 영상 피드구나"를 바로 이해해야 한다.

### 2. Pain Point

사용자가 지금 겪는 불편을 짧고 구체적으로 보여준다.

카드 후보:

- "인스타 부계에 올렸는데 다시 찾기 어렵다"
- "같은 문제 풀이 영상을 검색하기 어렵다"
- "내가 어느 난이도까지 풀었는지 영상으로 정리되지 않는다"

각 카드는 긴 설명보다 상황 중심 문장과 짧은 보조문으로 구성한다.

### 3. Core Flow

Limbing이 문제를 어떻게 해결하는지 3단계로 보여준다.

1. 영상을 올린다
2. 암장, 난이도, 문제를 태깅한다
3. 같은 문제를 푼 사람들의 영상을 모아본다

이 섹션은 랜딩페이지의 핵심이다. 제품 기능을 모두 나열하지 말고 "문제별로 영상이 모인다"는 구조를 이해시키는 데 집중한다.

### 4. Feature Preview

실제 앱이 있을 때 기대되는 화면을 미리 보여준다.

노출할 기능:

- 문제별 영상 피드
- 개인 기록 프로필
- 난이도/암장 필터
- 뱃지 또는 오늘의 Top 3

아직 실제 기능이 완성되지 않았으므로 `mock`, `preview`, `concept` 느낌이 나도 괜찮다. 다만 너무 허상처럼 보이지 않게 실제 사용 데이터를 닮은 텍스트를 넣는다.

### 5. Social Proof 또는 Research

문구 방향:

- 지금 클라이머들의 영상 기록 습관을 확인하고 있어요.
- 자주 가는 암장과 영상 업로드 방식을 알려주시면 초기 서비스 설계에 반영할게요.

수집 항목:

- 주로 가는 암장
- 클라이밍 영상 촬영 빈도
- 현재 공유하는 플랫폼
- 가장 보고 싶은 AI 기능

### 6. Final CTA

페이지 하단에서 사전 신청을 다시 유도한다.

구성 요소:

- 짧은 헤드라인
- 이메일 입력
- 제출 버튼

CTA 주변 문구는 부담을 줄이는 방향이 좋다.

## 8. 디자인 방향

### 전체 톤

- 모바일 영상 피드 중심
- 클라이밍 문화가 느껴지는 생동감
- 기록 서비스의 신뢰감
- 인스타그램보다 구조화되어 보이는 정리감

### 시각 요소

- 실제 제품 화면처럼 보이는 휴대폰 mock
- 문제 카드와 영상 썸네일
- 암장/난이도 필터 UI
- 뱃지와 랭킹 요소는 과하지 않게 보조로 사용

### 피해야 할 것

- 일반 SaaS 랜딩페이지처럼 보이는 추상 카드 나열
- 클라이밍 맥락이 없는 장식 이미지
- 기능 설명만 길게 이어지는 문서형 화면
- "AI", "커뮤니티", "플랫폼" 같은 큰 단어만 앞세우는 구성
