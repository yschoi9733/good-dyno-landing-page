# Limbing Landing Page Architecture

## 1. 목적

이 문서는 Good Dyno 팀이 만드는 Limbing 랜딩페이지 제작을 위한 설계 문서다.

랜딩페이지의 목적은 완성된 앱을 설명하는 것이 아니라, 클라이밍 영상을 찍고 공유하는 사용자에게 "이 서비스가 내 문제를 해결할 수 있겠다"는 감각을 빠르게 전달하고 사전 신청 또는 관심 등록으로 이어지게 만드는 것이다.

## 2. 서비스 한 줄 정의

Limbing은 같은 암장, 같은 난이도, 같은 문제를 푼 사람들의 클라이밍 영상을 한곳에 모아보는 영상 기반 클라이밍 기록 서비스다.

## 3. 랜딩페이지 타겟

초기 타겟은 클라이밍 영상을 반복적으로 촬영하고, 인스타그램 부계정이나 스토리로 풀이 과정을 공유하며, 막히는 문제를 풀기 위해 다른 사람의 영상을 찾아보는 사용자다.

숙련도보다 행동이 중요하다.

- 암장에서 본인 등반 영상을 자주 찍는다.
- 찍은 영상을 인스타그램, 스토리, 릴스, 단톡방 등에 올린다.
- 같은 문제를 다른 사람이 어떻게 풀었는지 궁금해한다.
- 내 풀이 영상과 성장 과정을 나중에 다시 보고 싶어한다.
- 클라이밍과 무관한 지인보다 클라이밍하는 사람들에게 영상을 보여주고 싶어한다.

## 4. 페이지 전체 흐름

랜딩페이지는 다음 순서로 사용자를 설득한다.

```
공감 → 이해 → 신뢰 → 전환
```

| 섹션 | 역할 |
|------|------|
| Nav | 어느 섹션에서든 CTA에 접근할 수 있게 한다 |
| Hero | 서비스가 무엇인지 한 문장으로 전달하고 첫 인상을 만든다 |
| PainPoints | 사용자가 겪는 불편을 짚어 공감을 끌어낸다 |
| CoreFlow | 서비스가 문제를 어떻게 해결하는지 단계별로 보여준다 |
| FeaturePreview | 구체적인 기능으로 서비스의 가치를 체감시킨다 |
| AIFeatures | AI 기능을 별도로 강조해 기술적 차별점을 전달한다 |
| SocialProof | 리서치 참여를 유도하며 신뢰감을 준다 |
| FinalCTA | 이메일 수집으로 전환을 완료한다 |

## 5. 섹션별 목적

### Hero

첫 화면에서 제품의 정체성이 바로 보여야 한다. 실제 앱 UI 목업을 통해 서비스 모습을 직접 보여주고, 사용자가 스크롤을 내릴 이유를 만들거나 바로 CTA로 연결한다.

### PainPoints

기능 설명보다 먼저 사용자가 이미 겪고 있는 불편을 건드린다. "맞아, 나도 이게 불편했어"라는 공감이 목적이다. 단색 디자인으로 문제의 무게감을 표현한다.

### CoreFlow

Limbing의 핵심 동작 방식을 3단계로 이해시킨다. 기능을 나열하지 않고, 영상 업로드 → 태깅 → 모아보기라는 흐름 자체를 전달하는 데 집중한다. 단계마다 다른 색상으로 구분해 긍정적인 인상을 준다.

### FeaturePreview

영상 아카이브, 필터, 영상 피드, 크루 기능을 비대칭 벤토 그리드로 소개한다. 각 기능마다 고유 액센트 컬러로 구분한다.

### AIFeatures

AI 컷편집, 클라이머 캐릭터화, 문제 풀이 추천, 자세 분석 피드백을 소개한다. Beta / Coming 상태를 뱃지로 구분해 출시 기대감을 높인다.

### SocialProof

"우리가 이미 리서치 중입니다"라는 신뢰감을 주면서 실제 유저 인사이트를 수집한다. 구글 폼과 연동된 설문 참여 자체가 또 다른 형태의 전환이다.

### FinalCTA

페이지를 끝까지 읽은 사람에게 마지막으로 행동을 유도한다. Formspree 연동 이메일 수집이 핵심 전환 지표다.

## 6. 기술 구조

### 프로젝트 구조

```
src/
├── components/
│   ├── Nav.tsx            # 네비게이션 (다크/라이트 토글, 스크롤 투탑)
│   ├── Hero.tsx           # 히어로 (앱 UI 목업 포함)
│   ├── PainPoints.tsx     # 문제 공감 (단색 카드)
│   ├── CoreFlow.tsx       # 이용 흐름 3단계 (컬러 리스트)
│   ├── FeaturePreview.tsx # 주요 기능 (벤토 그리드)
│   ├── AIFeatures.tsx     # AI 기능 (Beta/Coming 카드)
│   ├── SocialProof.tsx    # 유저 리서치 설문
│   └── FinalCTA.tsx       # 이메일 수집 폼
├── context/
│   └── ThemeContext.tsx   # 다크/라이트 모드 상태 관리
├── App.tsx
├── main.tsx
└── global.css             # Tailwind v4 테마 변수 및 글로벌 스타일
```

### 테마

`global.css`의 `@theme` 블록에 CSS 변수로 색상을 정의한다. 라이트 모드는 `:root[data-theme='light']`로 오버라이드한다. `ThemeContext`에서 `data-theme` 속성을 토글한다.

```
다크 모드 (기본)         라이트 모드
--color-bg: #0d0d0d     #f4f7f2
--color-accent: #4a7c59 (공통)
--color-accent-lime     라이트에서 진하게 오버라이드
--color-accent-yellow   라이트에서 진하게 오버라이드
```

### 스크롤

데스크탑(1024px 이상)에서 `scroll-snap-type: y mandatory`로 섹션 단위 스냅 스크롤이 적용된다. 모바일에서는 일반 스크롤로 동작한다.

### 외부 연동

| 서비스 | 용도 |
|--------|------|
| Formspree | 이메일 수집 (FinalCTA) |
| Google Forms | 유저 리서치 설문 (SocialProof) |
| Google Analytics (`gtag`) | 이메일 신청 완료 이벤트 트래킹 |

## 7. 디자인 방향

- 첫 화면에서 클라이밍 앱임을 시각적으로 바로 알 수 있어야 한다.
- 추상적인 카드 나열보다 실제 사용 맥락이 느껴지는 UI 목업을 사용한다.
- "AI", "커뮤니티", "플랫폼" 같은 큰 단어보다 구체적인 사용 상황을 앞세운다.
- 문제(PainPoints)는 단색으로, 해결(CoreFlow 이후)은 컬러로 대비를 준다.
